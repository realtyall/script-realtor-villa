# Threads 마케팅 전략 — 전자책 범용 구현 가이드

> 이 파일 하나로 어떤 전자책의 Threads 마케팅도 처음부터 끝까지 구현할 수 있다.
> 원본 적용 사례: 「컨설턴트 스크립트 50선」 (2026-04-16)

---

## 📌 이 파일의 사용법

1. **STEP 1** — 전자책 정보를 `[대괄호]` 항목에 채운다
2. **STEP 2** — 원본 파일(MD 또는 PDF)을 읽고 "콘텐츠 자산 발굴" 섹션을 채운다
3. **STEP 3** — 5가지 콘텐츠 유형 템플릿을 전자책 언어로 채워 초안을 작성한다
4. **STEP 4** — 주간 캘린더에 초안을 배정하고 게시한다
5. **STEP 5** — 반응을 기록하고 다음 달 콘텐츠에 반영한다

---

## 🗂️ PART 1. 전자책 기본 정보 파악

새 전자책에 적용할 때 아래 항목을 먼저 채운다.

```
전자책 제목: [                    ]
대상 독자: [                    ]
독자의 핵심 고통: [              ]
  — 고통 1: 
  — 고통 2: 
  — 고통 3: 
독자의 핵심 욕구: [              ]
책의 구성 단계 수: [  ]단계
각 단계 이름:
  — 1단계:
  — 2단계:
  — 3단계:
  (이하 계속)
허브페이지 URL: [                ]
구매 Google Form URL: [          ]
```

## 🎯 PART 2. 타깃 독자 프로파일 작성법

아래 질문에 답하면 타깃이 명확해진다.

| 질문 | 답변 |
|---|---|
| 이 책을 사는 사람의 직업·역할은? | |
| 경력 몇 년차가 가장 절박한가? | |
| 하루 중 가장 힘든 순간이 언제인가? | |
| 어떤 말을 들으면 가슴이 철렁 내려앉는가? | |
| 어떤 말을 들으면 가슴이 뚫리는 것 같은가? | |
| 이 책을 사면 얻는 가장 구체적인 변화는? | |

> **원칙:** 타깃이 구체적일수록 게시물이 강해진다.
> "직장인"이 아니라 "입사 2년차 영업 사원"처럼 좁혀라.

## 📚 PART 3. 원본에서 콘텐츠 자산 발굴

전자책 원본(MD, PDF)을 읽으며 아래 항목을 채운다.

### 3-1. 훅 문장 수집 (최소 10개)

원본에서 강하게 와닿는 문장을 그대로 옮긴다.
이 문장들이 Threads 첫 줄(훅)로 직접 사용된다.

```
훅 문장 1: "                              "
훅 문장 2: "                              "
훅 문장 3: "                              "
훅 문장 4: "                              "
훅 문장 5: "                              "
훅 문장 6: "                              "
훅 문장 7: "                              "
훅 문장 8: "                              "
훅 문장 9: "                              "
훅 문장 10: "                             "
```

> **어디서 찾을까?**
> - 머리말 / 에필로그 (감성 언어가 집중됨)
> - 각 챕터 "들어가며" 첫 단락
> - 독자의 고통을 묘사한 문장
> - 역설처럼 들리는 원칙 문장

### 3-2. ❌/✅ 비교 쌍 수집 (최소 5쌍)

"잘못된 방식 vs 올바른 방식"을 원본에서 찾는다.
책의 스크립트, 예시 대화, Before/After 등을 활용.

```
비교 쌍 1:
  ❌ 잘못된 방식: "                        "
  ✅ 올바른 방식: "                        "
  이유 (한 줄): 

비교 쌍 2:
  ❌ 잘못된 방식: "                        "
  ✅ 올바른 방식: "                        "
  이유 (한 줄): 

비교 쌍 3:
  ❌ 잘못된 방식: "                        "
  ✅ 올바른 방식: "                        "
  이유 (한 줄): 

비교 쌍 4: ...
비교 쌍 5: ...
```

### 3-3. 통념 반전 목록 (최소 5개)

독자가 당연하다고 여기는 생각을 뒤집는 원칙.
"사실 반대다" "역설이지만 맞다" 형태.

```
반전 1: [통념] _________________ → [사실] _________________
반전 2: [통념] _________________ → [사실] _________________
반전 3: [통념] _________________ → [사실] _________________
반전 4: [통념] _________________ → [사실] _________________
반전 5: [통념] _________________ → [사실] _________________
```

### 3-4. 핵심 원칙 목록 (최소 5개)

책 전체를 관통하는 본질적 인사이트.
챕터 소제목, 강조 문장, 박스 텍스트에서 찾는다.

```
원칙 1: 
원칙 2: 
원칙 3: 
원칙 4: 
원칙 5: 
```

## ✍️ PART 4. 5가지 콘텐츠 유형과 작성 공식

모든 게시물은 아래 5가지 유형 중 하나다.
유형별로 다른 심리 → 다른 행동(팔로우, 저장, 공유, 클릭)을 유도한다.

---

### 유형 A — 공감형 | 목적: 팔로우 유도

**언제 쓰는가:** 월요일. 독자의 고통을 정면으로 묘사할 때.

**작성 공식:**
```
[훅] 독자가 경험하는 고통의 장면을 구체적으로 그린다 (3~5줄)
[전개] 그 고통이 왜 생기는지 1~2줄
[반전] 준비·방법·관점이 달라지면 바뀐다는 사실
[CTA] "저장해두세요" 또는 가벼운 링크
```

**작성 원칙:**
- 첫 줄이 독자의 가슴을 때려야 한다. "이거 나 얘기다"가 느껴져야 한다.
- 판매 문구를 쓰지 않는다. 공감이 먼저다.
- 링크는 있어도 되고 없어도 된다. 목적은 팔로우다.

**템플릿 (채워 넣기):**
```
[독자 직업]으로 [기간]이 지났는데
아직도 [고통의 장면]이 반복된다면.

[그 상황이 왜 힘든지 1~2줄].

이건 [능력/재능/운]의 차이가 아니다.
[준비/방법/접근법]의 차이다.

📌 저장해두고, [다음 기회]에 꺼내보세요.
```

---

### 유형 B — 장면형 | 목적: 저장 유도

**언제 쓰는가:** 화요일·토요일. 현장에서 바로 쓸 수 있는 말을 보여줄 때.

**작성 공식:**
```
[상황] 독자가 자주 마주치는 그 장면 1줄
[잘못된 말] ❌ 표시와 함께 대부분이 하는 방식
[침묵 또는 결과] 그 말의 효과 없음 묘사
[올바른 말] ✅ 표시와 함께 대안 표현
[이유] 왜 이 말이 통하는지 2~3줄 (심리 설명)
[CTA] "이런 [방법]이 [숫자]개 있습니다 → 링크"
```

**작성 원칙:**
- ❌/✅ 대비 구조가 이 유형의 핵심이다. 비교가 없으면 약해진다.
- 이유 설명은 전문 용어 없이. 독자가 이해하면 저장한다.
- 게시물 자체로 완결돼야 한다. 링크 없이도 가치 있어야 한다.

**템플릿 (채워 넣기):**
```
[상황 묘사]에서 이렇게 [말/행동]하고 있다면.

❌ "[잘못된 방식]"

[그 방식의 문제점 1~2줄].

이렇게 바꿔라.

✅ "[올바른 방식]"

[왜 이 방식이 통하는지 2~3줄].

📎 이런 [방법/스크립트/원칙]이 [숫자]개 → [링크]
```

---

### 유형 C — 반전형 | 목적: 공유·저장 유도

**언제 쓰는가:** 목요일. 통념을 뒤집는 날카로운 관점을 제시할 때.

**작성 공식:**
```
[통념] 독자가 당연하게 여기는 생각 or 자주 듣는 말 1~2줄
[반전 선언] "사실은 반대다" 또는 역설적 사실
[근거] 왜 통념이 틀렸는지 혹은 더 정확한 사실
[구체적 방법] 그러면 어떻게 해야 하는가
[CTA] 가벼운 링크
```

**작성 원칙:**
- 첫 줄이 "어? 그게 틀렸어?"를 유발해야 공유가 일어난다.
- 반박이 아니라 더 정확한 시각을 제시하는 방식으로 쓴다.
- 결론이 실용적이어야 한다. 인사이트만 있고 적용법이 없으면 약하다.

**템플릿 (채워 넣기):**
```
"[독자가 흔히 듣거나 믿는 통념]"

이 말이 [절반/완전히] [맞다/틀렸다].

[왜 통념이 불완전한지 2~3줄].

[더 정확한 사실 또는 반전 원칙].

[실제로 어떻게 해야 하는지 1~2줄].

📎 [책 제목] → [링크]
```

---

### 유형 D — 원칙형 | 목적: 신뢰 축적

**언제 쓰는가:** 수요일. 깊이 있는 인사이트로 전문성을 보여줄 때.

**작성 공식:**
```
[선언] 강력한 원칙 한 줄
[설명] 그 원칙이 왜 중요한지 3~5줄
[현장 적용] 실제로 이 원칙이 어떻게 작동하는지
[결론] 한 줄 요약
[CTA] 없어도 되고 있어도 됨
```

**작성 원칙:**
- 선언 문장이 강해야 한다. "~해야 한다" "~이 전부다" 형태가 효과적이다.
- 링크를 억지로 붙이지 않는다. 이 유형의 목적은 신뢰 축적이다.
- 읽고 나서 "맞아, 그렇지"가 느껴지면 성공이다.

**템플릿 (채워 넣기):**
```
[핵심 원칙을 한 줄로].

[그 원칙이 중요한 이유 2~3줄].

[현장/실제 상황에서 이 원칙이 작동하는 방식 2줄].

[결론 한 줄].

📎 [선택적 링크]
```

---

### 유형 E — CTA형 | 목적: 구매 전환

**언제 쓰는가:** 금요일. 허브페이지 클릭과 구매를 유도할 때.

**작성 공식:**
```
[공감] 독자의 고통을 2줄로 요약
[가치 제시] 이 책이 주는 구체적인 것
[증거] 구성·분량·수량 등 구체적 숫자
[행동 유도] 링크 + 명확한 지시
```

**작성 원칙:**
- "사세요"라는 말을 쓰지 않는다.
- 링크 앞에 반드시 구체적인 행동 지시가 있어야 한다. ("확인하세요", "상세 보기")
- 목차를 그대로 노출하는 것이 가장 효과적인 CTA 유형 중 하나다.

**템플릿 (채워 넣기):**
```
[독자의 가장 큰 고통 1줄].

[그 고통이 아직 해결 안 된 이유 1~2줄].

[책의 목차 or 구성 요소 나열]:
  — [챕터명 1]
  — [챕터명 2]
  — [챕터명 3]
  (이하)

[총 분량/스크립트/원칙 수] — 구체적 숫자로.

👇 [행동 지시]
[링크]
```

## 📅 PART 5. 주간 운영 캘린더 (복사해서 사용)

| 요일 | 유형 | 목적 | 비고 |
|---|---|---|---|
| 월요일 | A. 공감형 | 팔로우 유도 | 주초 감성 |
| 화요일 | B. 장면형 | 저장 유도 | ❌/✅ 대비 |
| 수요일 | D. 원칙형 | 신뢰 축적 | 깊이 있는 글 |
| 목요일 | C. 반전형 | 공유 유도 | 주중 피크 |
| 금요일 | E. CTA형 | 구매 전환 | 주말 결정 공략 |
| 토요일 | B. 장면형 (가벼운 버전) | 저장 유도 | 현장 활용 |
| 일요일 | 휴식 or A. 공감형 | 다음 주 예열 | 선택적 |

**권장 게시 시간:**
- 오전 7:30~8:30 (출근 준비)
- 오후 12:00~13:00 (점심)
- 오후 21:00~22:00 (귀가 후)

## 📋 PART 6. 5주 시리즈 콘텐츠 플랜

책의 구성 단계를 주차별로 전개한다.
매 주 말(금요일) CTA 게시물로 허브페이지를 연결한다.

```
1주차 테마: [1단계 이름] — "[1단계를 한 줄 요약한 슬로건]"
  - 월: A. 공감형 — [1단계의 핵심 고통 묘사]
  - 화: B. 장면형 — [1단계 핵심 스크립트/방법 ❌/✅]
  - 수: D. 원칙형 — [1단계의 핵심 원칙]
  - 목: C. 반전형 — [1단계와 관련된 통념 반전]
  - 금: E. CTA형 — "[1단계 포함" 언급 + 링크]

2주차 테마: [2단계 이름] — "[2단계를 한 줄 요약한 슬로건]"
  (동일 구조 반복)

3주차 테마: [3단계 이름]
4주차 테마: [4단계 이름]
5주차 테마: [5단계 이름] + 전체 CTA 집중 주간
```

## 🪝 PART 7. 훅 패턴 6가지

어떤 전자책에도 적용 가능한 훅 패턴이다.
각 패턴에 맞는 문장을 PART 3에서 가져와 채운다.

| 패턴 | 구조 | 예시 (컨설턴트 스크립트 50선) |
|---|---|---|
| **고통 직구** | 독자의 고통 장면을 직접 묘사 | "처음 픽업 나가던 날, 어색한 침묵을 채우려 라디오 볼륨만 높였다." |
| **숫자 충격** | 구체적 수치로 관심 유발 | "스크립트 50개. 심리기제 설명 50개. 거절 시나리오 10개." |
| **질문 던지기** | 독자가 스스로 답하게 만들기 | "고객이 '좀 더 생각해볼게요' 할 때, 당신은 뭐라고 하는가?" |
| **역설 제시** | 통념을 뒤집는 한 마디 | "집을 팔지 않겠다고 선언할 때 계약률이 올라간다." |
| **인용 활용** | 책 속 강력한 문장 그대로 | "집은 사는 것이 아니라 인연이 닿아 주인이 되는 것이다." |
| **before/after** | 이전 vs 이후 대비 | "❌ '남향입니다' → ✅ '아침에 햇살 받으면서 커피 한 잔 마시면'" |

## 👤 PART 8. 프로필 바이오 작성 공식

### 바이오 구조
```
[직업/역할] 한 줄
[하는 일의 본질] 한 줄
[독자와 공유하는 경험 또는 전환점] 한 줄
[전자책 소개 + 링크]
```

### 3가지 톤 버전

**공감 중심 버전 (권장)**
```
[직업]입니다.
[현장에서 하는 일을 구체적으로].
"[과거의 고통 또는 배움]"
그 [방법/말/원칙]을 정리했습니다 👇
[링크]
```

**전문성 중심 버전**
```
[직업] [경력]
[하루를 한 문장으로 묘사].
현장에서 통하는 말, 안 통하는 말을 씁니다.
[전자책 제목] 👇
[링크]
```

**초간결 버전**
```
[직업]의 현장 언어를 씁니다.
```

## 📏 PART 9. 게시물 공통 규칙

어떤 전자책에도 동일하게 적용하는 규칙이다.

### 형식 규칙
- **줄 바꿈**: 2~3줄마다. 모바일에서 읽기 편하게.
- **이모지**: 게시물당 최대 2개. 남발하면 신뢰가 떨어진다.
- **해시태그**: 달지 않거나 1~2개만.
- **CTA**: 게시물당 딱 하나. 링크 두 개 이상은 금지.
- **길이**: 5~15줄이 최적. 20줄 넘으면 이탈이 높아진다.

### 언어 규칙
- "사세요", "구매하세요"는 쓰지 않는다.
- AI 티가 나는 문장을 제거한다:
  - ❌ "오늘날의 급변하는 환경에서"
  - ❌ "이를 통해 시너지를 극대화하여"
  - ❌ "탁월한 성과를 달성할 수 있습니다"
- 독자가 현장에서 쓰는 언어로 쓴다.
- 마침표를 쓰지 않는 문장 단위 줄 바꿈이 Threads에 잘 맞는다.

### 저장 유도 문구 (돌아가며 사용)
```
📌 저장해두고, [상황]에 꺼내보세요.
나중에 찾을 수 있게 저장하세요.
📎 [전체 목록/스크립트] → [링크]
```

### 댓글 유도 문구 (월 1~2회)
```
여러분은 이 상황에서 어떻게 하세요?
이 [방법/말], 실제로 써봤다면 댓글 남겨주세요.
```

## 📊 PART 10. 성과 측정 및 개선

### 측정 지표

| 지표 | 기준 | 측정 방법 |
|---|---|---|
| 팔로워 증가 | 주 +[목표]명 | Threads 기본 통계 |
| 저장율 | 좋아요 대비 15~20% | 게시물 통계 |
| 링크 클릭 | 게시물당 [목표]회 | 허브페이지 UTM 파라미터 |
| 허브→구매 전환 | 클릭 대비 3~5% | Google Form 유입 확인 |

### UTM 파라미터 설계
허브페이지 링크에 아래 파라미터를 붙여 어떤 게시물이 클릭을 만드는지 추적한다.

```
기본 URL: https://[허브페이지주소]
파라미터:
  utm_source=threads
  utm_medium=social
  utm_campaign=[전자책코드]
  utm_content=[게시물유형코드]

예시:
  A.공감형 → utm_content=empathy
  B.장면형 → utm_content=scene
  C.반전형 → utm_content=reversal
  D.원칙형 → utm_content=principle
  E.CTA형 → utm_content=cta
```
